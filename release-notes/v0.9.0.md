# What's New in Version 0.9.0

**Released:** April 23, 2026

## 🛡️ Security & Data Privacy

### Customer Data Is Never Stored by AI

TIP Generator now explicitly signals Anthropic to skip retention on every API call. Your customer documents are processed for generation and immediately discarded — not logged, not retained, not used for training.

**What changed:**
- All Claude API requests include `X-Anthropic-Do-Not-Store: true`
- New **Your Data is Protected** badge on the home screen
- Published `SECURITY.md` documenting the full data flow for audit trail

---

## 🔒 PII Pseudonymization (Opt-In)

### Scrub Customer Identifiers Before Sending to Claude

You can now enable PII scrubbing when generating a TIP. When enabled, sensitive customer identifiers are replaced with tokens before the document text leaves your infrastructure — then automatically restored in the final TIP.

**What gets scrubbed:**
- IP addresses, MAC addresses, subnets/CIDRs
- Hostnames, FQDNs, domain names
- Email addresses, phone numbers
- Person names, company/organization names, locations
- VLANs and other network identifiers

**How it works:**
1. On the Generate page, check **Scrub customer PII before sending to Claude**
2. Customer identifiers are replaced with tokens like `{{IP_ADDRESS_1}}`, `{{PERSON_1}}`
3. Claude generates the TIP using only the tokens
4. Tokens are restored to original values before the TIP is saved
5. The mapping is stored securely in the database, tied to your draft

**Status:** Opt-in today. Will become always-on after quality verification.

## 🧩 Phase 2.5 — Section-Chunk Playbook (RAG)

### Thrive Playbook Content Injected at Generation Time

Approved library documents are now chunked into section-level pieces and matched to the relevant sections of your TIP using BM25 keyword retrieval. Claude receives authoritative Thrive playbook content alongside your customer data.

**What this means:**
- Standard Thrive processes, naming conventions, and steps are woven into your TIP automatically
- No external embedding API required — retrieval runs fully on-prem
- 193 chunks indexed from 11 approved library documents
- New chunks are indexed automatically when a library document is approved
