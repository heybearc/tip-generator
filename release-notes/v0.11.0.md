# What's New in Version 0.11.0

**Released:** 2026-04-24

---

## ✨ New Features

### Fill [DATA NEEDED] Fields — No Claude Required
After generating a TIP, you'll often see orange `[DATA NEEDED: ...]` markers where the source documents didn't have enough detail — things like a project manager name, a specific IP address, or a go-live date.

You can now fill these in directly inside TIP Generator:

1. Open any completed draft. If DATA NEEDED fields exist, an orange **Fill (N)** button appears in the toolbar — the number shows how many unique fields were found across all sections.
2. Click it to open the panel.
3. Enter values for the fields you know. Leave the rest blank — they'll be skipped.
4. Click **Apply All**. Every section updates instantly and saves automatically.

No Claude call, no tokens burned. Run it multiple times as you gather information.

---

### Author Instructions & Presets
You can now give Claude writing guidance before generation starts — controlling tone, depth, and document style on a per-generation basis.

**How to use it:**
- Find the **Author Instructions** field on the Generate page.
- Type any instruction (e.g. "Write for a project manager audience, high-level only").
- Optionally save it as a **Preset** so you can load it with one click next time.

Presets are saved to your profile and available across all sessions.

---

### Cancel a Running Generation
A **Cancel** button now appears directly on the Drafts page next to any generating draft. Click it to stop the job immediately. The draft moves to a new **cancelled** status (orange badge) and keeps whatever content was completed up to that point.

---

### Promote Sections to Library
From any completed draft section, you can now **promote** it to the TIP Library. Promoted sections appear as reusable chunks on the Library page, visible per document card. Use this to build a library of strong boilerplate sections for common project patterns.

---

## 🔧 Reliability Improvements

- **Parallel generation** — the worker now runs up to 3 generations simultaneously without jobs blocking each other.
- **Smarter restart recovery** — if the system restarts mid-generation, it now waits 10 minutes before deciding a task is truly stuck, preventing false re-queues during routine deployments.
- **Thread-safe AI initialization** — fixed a rare race condition where two simultaneous generations could conflict during startup.
- **Cancel is now instant** — cancellation sends an immediate stop signal rather than waiting for the current batch to finish.

---

## 💡 What You Need to Know

- No action required. All features are available immediately after this release.
- The Fill button only appears on completed drafts that contain `[DATA NEEDED]` markers. If all fields are filled, the button disappears.
- Author Instructions are optional and do not affect existing drafts — only new generations.
- Cancelled drafts are kept permanently and can still be exported with whatever content was generated.
